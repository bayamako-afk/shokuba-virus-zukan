# Season 1 2プロジェクト命名ルール

## 1. 目的

本ルールは、**各EPをアニメ版と実写版の2プロジェクトで並走させる際に、命名の混線を防ぐための標準規則**を定義する。今後は同一EPに対して複数の制作物が並ぶため、話数・ウイルス名・表現レーン・文書役割が一目で分かる命名を優先する。

## 2. 基本原則

命名では、**EP番号を先頭に置き、その後にウイルス名またはテーマ識別子、最後にレーン種別を置く**。これにより、一覧表示でも話数順に並びやすくなり、アニメ版と実写版の対応関係もすぐに判別できる。運用上は、英小文字・アンダースコア区切りを基本とし、レーン名は固定語を用いる。

| 項目 | ルール |
|---|---|
| EP番号 | `EP001` のように3桁ゼロ埋め |
| 識別子 | `tameiki_virus` のように英小文字 + アンダースコア |
| レーン名 | `anime` または `liveaction` に固定 |
| 区切り | すべてアンダースコア `_` を使用 |
| 禁止 | 日本語・空白・ハイフン混在・略称ぶれ |

## 3. 推奨命名形式

### 3-1. プロジェクト名

Zopiaや管理表で最も基準になるのは、以下の形式である。

| 用途 | 推奨形式 | 例 |
|---|---|---|
| アニメ版プロジェクト | `EP{番号}_{virus_key}_anime` | `EP001_tameiki_virus_anime` |
| 実写版プロジェクト | `EP{番号}_{virus_key}_liveaction` | `EP001_tameiki_virus_liveaction` |

この形式を正式採用する。日本語表示が必要な場面では別途表示名を使ってよいが、**内部管理名は英字形式に統一**する。

### 3-2. 日本語表示名

人間向けの確認用として、日本語表示名を併記する場合は以下を使う。

| 用途 | 日本語表示例 |
|---|---|
| アニメ版 | `EP001_ため息ウイルス_アニメ版` |
| 実写版 | `EP001_ため息ウイルス_実写版` |

ただし、GitHub上のファイル名、管理キー、プロジェクト識別子では英字形式を優先する。

## 4. 文書命名ルール

EP単位の文書は、**共通文書** と **レーン別文書** を明確に分ける必要がある。共通脚本やshotlist骨子はEP共通名で管理し、アニメ版・実写版で内容が分岐する文書だけにレーン識別子を付ける。

| 文書種別 | 命名方針 | 例 |
|---|---|---|
| 共通脚本 | EP共通名 | `EP001_ため息ウイルス.md` |
| 共通構成資料 | EP共通名 + 文書役割 | `EP001_試験編集版_構成指示書.md` |
| アニメ版運用文書 | EP番号 + レーン + 役割 | `EP001_アニメ版_Zopiaプロンプト.md` |
| 実写版運用文書 | EP番号 + レーン + 役割 | `EP001_実写版_Zopia投入手順書.md` |

## 5. アセット・フォルダ命名への接続

既存のアセット命名規則である **英小文字・アンダースコア区切り・version付き** は維持する。今回の2レーン化では、**アセット自体をアニメ版 / 実写版で無理に二重命名しない** ことが重要である。基準素材は主に実写版レーンで使い、アニメ版は必要最小限の参照に留める。

| 対象 | 方針 |
|---|---|
| 共通ロゴ・UI | 既存命名を維持する |
| 実写版基準素材 | 既存命名を維持し、出荷箱で整理する |
| アニメ版参照素材 | 専用素材が必要になるまで増やしすぎない |

## 6. 管理表キーのルール

管理表では、1行を1EPとし、その中に **anime / liveaction** の列を持たせる。列名や内部キーは、今後の自動化を見据えて短く固定する。

| 管理対象 | 推奨キー |
|---|---|
| アニメ版進捗 | `anime_status` |
| 実写版進捗 | `liveaction_status` |
| 実写基準素材状況 | `liveaction_asset_status` |
| 初回生成結果 | `first_gen_result` |
| 公開状況 | `release_status` |

## 7. 避けるべきぶれ

今後もっとも避けるべきなのは、`live`, `live_action`, `real`, `photo` などの表記ゆれである。レーン名は **`anime`** と **`liveaction`** に固定する。また、ウイルス名キーも都度揺らさず、最初に決めた英字キーを継続使用する。

## 8. ひとことでまとめ

プロジェクト名は **`EP{番号}_{virus_key}_anime`** と **`EP{番号}_{virus_key}_liveaction`** を正式採用し、文書・管理表・運用メモもこの二分法に揃える。共通脚本は共通名、分岐文書だけレーン識別子を付ける、という整理を標準とする。
