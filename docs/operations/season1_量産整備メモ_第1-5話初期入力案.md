# 職場ウイルス図鑑｜Season 1量産整備メモ（所見・最小修正案・第1〜5話初期入力案・次アクション）

## 1. 所見
添付された4本のMarkdownは、**Season 1 を「違和感の発見と命名」に限定し、反撃前までで止める**という方針で、全体として十分に整合している。特に、`season1_episode_sheet_template.md` が各話の核を固定し、`short_template_15_30_45.md` が尺別の量産フォーマットを持ち、`virus_registry.md` が命名の重複防止と昇格候補管理を担い、`character_registry.md` が守谷守を**固定絶対主人公ではなく代表観測者**として整理している点は、Season 1 の量産思想と一致している。

また、4本の文書はそれぞれ役割が重なりすぎておらず、**企画・尺・命名・人物型**が分離されているため、制作管制の土台として扱いやすい。とくに、Season 2 以降の拡張余地を残したまま、Season 1 ではAI相棒や自己防衛隊を匂わせに留める設計になっていることは、シリーズ全体の進化設計とも噛み合っている。

一方で、量産フェーズへ入る直前の状態として見ると、**不足は大きくないが、回転を落としやすい小さな抜け**がいくつかある。現状のテンプレートは企画としては十分だが、実務上の前処理としては、**どの回を先に作るか、どの尺を主力にするか、命名演出をどの型で固定するか、どの人物資産を再利用するか**がまだ少し散っている。したがって、今必要なのは大幅改修ではなく、**量産を止めない最小限の補強**である。

| 確認項目 | 現状評価 | 所見 |
|---|---|---|
| Season 1 の目的 | 良好 | 「違和感の発見と命名」で統一されている |
| 守谷守の位置づけ | 良好 | 代表観測者として一貫している |
| 尺運用 | 良好 | 15秒 / 30秒 / 45秒の役割が明確 |
| ウイルス命名管理 | 良好 | 台帳化されており重複防止に有効 |
| 人物量産設計 | 良好 | 型管理が機能している |
| 実務の回転性 | 要微補強 | 量産順・演出固定・再利用資産欄が薄い |

## 2. 最小修正案
今回の前提が**深さよりまず回転重視**である以上、修正は最小でよい。新しい大テンプレートを増やすより、既存の4本に**3つだけ短い補助欄**を足す方が効率的である。

第一に、`season1_episode_sheet_template.md` には、各話の終わり方を量産しやすくするため、**命名演出パターン**を固定する欄を追加するとよい。現状でも命名演出は書けるが、`図鑑登録型 / AI観測ログ型 / 断定型` のいずれを採るかを明示できると、毎話で迷いにくい。第二に、**再利用資産キー**を1行だけ加えるとよい。たとえば `office_morning_common / approval_desk_common / meeting_room_A / ai_log_overlay_v1` のように、背景やUIの流用先が見えるようになる。第三に、**主力尺**を1つだけ固定する欄を設けるとよい。現在は15秒 / 30秒 / 45秒の候補を書けるが、量産時は「今回の正本は30秒、15秒は抜粋、45秒は保留」のように主力を決めておいた方が工程が軽い。

`virus_registry.md` については、大きな改修は不要である。ただし、`notes` とは別に **launch_priority** を `A / B / C` で持たせると、最初の10本の優先順位を決めやすい。Season 1 量産初期は、刺さりやすさ、撮りやすさ、演出の見えやすさで優先順位を切る必要があるため、この1列だけで実務はかなり軽くなる。

`character_registry.md` については、型設計は十分であるが、**初期5話で実際に使う人物セット**を明示するとよい。つまり、「守谷守」「上司圧力型」「同調周囲型」「無自覚加害型」「被感染者型」のうち、どれを初期5話で回すかだけを先に固定する。これにより、image2 の人物固定も最小回数で済む。

| 文書 | 最小補強内容 | 目的 |
|---|---|---|
| episode_sheet | 命名演出パターン欄 | ラストの量産性を上げる |
| episode_sheet | 再利用資産キー欄 | 背景・UI流用を明示する |
| episode_sheet | 主力尺欄 | 15/30/45の迷いを減らす |
| virus_registry | launch_priority列 | 先に作る回を迷わないようにする |
| character_registry | 初期5話使用人物セット | image2固定対象を絞る |

この補強で十分であり、**新しい管理資料を増やす必要はまだない**。現段階では「管理を良くする」より「回せる状態にする」ことを優先すべきである。

## 3. 第1〜5話初期入力案
以下では、優先候補として指定された5本を、**Season 1量産の初期入力版**として整理する。いずれも30秒を主力尺にし、15秒は先行出しまたは抜粋、45秒は当たり回の試作拡張という扱いを前提にしている。

## 4. 第1〜5話の並び順に関する所見
指定された5本の候補順は妥当であるが、**視聴者にとっての入りやすさ**と**制作の回しやすさ**の両方を考えると、次の順番がもっとも安定する。まず、空気だけで成立する「ため息ウイルス」で世界観を開き、次に反復の不気味さがある「差し戻し確認ウイルス」でシリーズの型を見せる。その後に「丸投げウイルス」で人間関係の怖さを入れ、「口頭指示ウイルス」でSeason 2 接続の芽を示し、最後に「責任なすりつけウイルス」で構造っぽさとオムニバス性を残す。

| 推奨順 | 話数 | 理由 |
|---|---|---|
| 1 | ため息ウイルス | 一撃で世界観が伝わる |
| 2 | 差し戻し確認ウイルス | Season 1 の型を定着させやすい |
| 3 | 丸投げウイルス | 共感コメントを取りにいける |
| 4 | 口頭指示ウイルス | 防衛OSへの芽を匂わせやすい |
| 5 | 責任なすりつけウイルス | オムニバス性と構造感が立つ |

## 5. 次アクション
次工程では、深掘りよりも**制作が止まらない順序**が重要である。したがって、実務は以下の順で進めるのがよい。まず、5本分の `episode_sheet` を今回の初期入力案ベースで埋める。次に、5本に共通する再利用資産キーを仮固定する。続いて、守谷守、上司圧力型、同調周囲型、無自覚加害型、若手事務スタッフの**最小人物セット**だけを確定する。そのうえで、各話を30秒の主力尺で `shotlist` 化し、最後に15秒抜粋版の切り出し位置だけ決める。この順番なら、映像生成前の判断がほぼ終わる。

| 順序 | 次アクション | 成果物 |
|---|---|---|
| 1 | 5本分の episode sheet を記入版へする | EP001〜EP005 シート初稿 |
| 2 | 命名演出パターンを各話で固定する | ending rule 一覧 |
| 3 | 再利用資産キーを仮固定する | background / UI 流用表 |
| 4 | 初期5話で使う人物セットを固定する | character fix list |
| 5 | 各話30秒版の shotlist を作る | shotlist_vertical 初稿 |
| 6 | 15秒抜粋版の切り出し位置を決める | short cutdown memo |
| 7 | その後にだけ image2 と映像工程へ進む | 最小実行の制作開始 |

回転重視という条件に照らすと、**次にやるべき実務は「第1〜5話の記入版 episode sheet 化」**である。ここを確定すれば、その後の人物固定、ショット設計、映像化の迷いが一気に減る。

## 6. 結論
現状の添付MD一式は、Season 1量産フェーズに入るための土台として十分に強い。足りないのは大きな再設計ではなく、**量産のための微細な固定**だけである。したがって、今のフェーズでは新資料を増やすより、今回整理した**5本の初期入力案をそのまま各話シートに流し込む**のが最短である。

次段階として最も自然なのは、**EP001〜EP005 の記入版 episode sheet を実作成すること**であり、その際にあわせて `launch_priority`、命名演出パターン、再利用資産キーを埋めることで、Season 1量産ラインはかなり実務化できる。
