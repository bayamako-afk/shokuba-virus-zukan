# 職場ウイルス図鑑 Season1 1話1フォーム診断項目

## 1. 文書の目的

本書は、『職場ウイルス図鑑 Season1』の EP001〜EP010 について、各話ごとに**1話1フォーム・3問構成**の簡易チェックを実装できるようにするための運用文書である。目的は、視聴者が動画視聴後に気軽に参加できる**職場あるあるチェック**を用意し、コメント、概要欄、外部フォーム、将来の「職場ウイルス度チェック」本体へ接続しやすい診断素材として整備することにある。[1] [2]

本書で扱うフォームは、医療的、心理学的、メンタルヘルス的な診断ではない。あくまで創作コンテンツ連動の軽い参加施策として設計し、結果も断定ではなく**「潜んでいるかもしれません」**という余白を残した表現へ統一する。[1]

## 2. 共通運用ルール

Season1 は「発見編」であり、視聴者に説明しすぎず、直感的に「あるある」と感じてもらうことを優先する。そのため、各フォームも回答時間30秒以内を想定し、**1フォーム3問**、**回答は3択のみ**、**結果表示は3段階**で統一する。[1] [2]

| 項目 | 共通ルール |
|---|---|
| フォーム数 | 1話につき1フォーム |
| 設問数 | 1フォーム3問 |
| 回答時間想定 | 30秒以内 |
| 回答形式 | **よくある / たまにある / ほとんどない** |
| 採点ルール | よくある = 2点 / たまにある = 1点 / ほとんどない = 0点 |
| 結果表示 | 0〜1点 = 潜伏レベル / 2〜4点 = 感染注意レベル / 5〜6点 = 職場ウイルス反応あり |
| 話数別の最後の一言 | 各EPごとに1文だけ変え、SNSで拾いたくなる余韻を残す |
| 表現制限 | 「ストレス診断」「メンタル診断」などの表現は使わない |
| 結果文体 | 断定せず「潜んでいるかもしれません」に留める |
| 必須表記 | EP番号・ウイルス名・型を残す |

## 3. 採点ルールの共通表記

各フォームでは、次の採点ルールを共通表示として用いる。

| 回答 | 点数 |
|---|---|
| よくある | 2点 |
| たまにある | 1点 |
| ほとんどない | 0点 |

また、結果表示は次の3段階で共通化する。

| 0〜1点 | 潜伏レベル |
| 2〜4点 | 感染注意レベル |
| 5〜6点 | 職場ウイルス反応あり |

## 4. Season2フォーム拡張方針

Season2では、Season1の軽さを壊さない範囲で、各ウイルスの**株ごとの傾向**を把握するため、フォームを3問から**5問程度**へ拡張する。Season1が「まず参加してもらう」ための入口であるのに対し、Season2は「同じウイルスでもどの株が出やすいか」を、視聴者参加型で少しだけ深く観測する段階として位置づける。[1] [2]

Season2で追加するのは、原則として**株の表示**と**任意の職場規模項目**の2点のみとする。年代、性別、地域などの属性情報はSeason2では取得せず、まずは参加率と回答しやすさを優先する。IPによる地域推定も使わず、引き続き**創作コンテンツ連動の職場あるあるチェック**として運用する。

| 項目 | Season1 | Season2 |
|---|---|---|
| 設問数 | 3問 | 5問程度 |
| 株 | 非表示 | 表示 |
| 職場規模 | なし | 任意 |
| 年代 | 取得しない | 取得しない |
| 性別 | 取得しない | 取得しない |
| 地域 | 取得しない | 取得しない |
| IP地域推定 | 使わない | 使わない |

### Season2の目的

Season2は「変異編」として、同じウイルスでも**どのような株が出やすいか**を視聴者参加型で把握する段階とする。ただし、データ収集を前面に出しすぎず、あくまで**創作コンテンツ連動の職場あるあるチェック**としての軽さを維持する。

また、Season全体のフォーム設計は次の段階目標で整理する。

| Season | 目的 |
|---|---|
| Season1 | 気軽に参加してもらう |
| Season2 | 株ごとの傾向を見る |
| Season3 | ワクチン・防衛策へつなげる |
| Season4 | 職場ウイルス分布図へ展開する |

## 5. 運用上の注意

フォーム実装時は、タイトル、結果表示、概要欄文のどこでも**診断の断定性を上げすぎない**ことが重要である。特に「あなたは感染しています」「職場が危険です」などの断定表現は避け、必ず**創作コンテンツ連動の職場あるあるチェック**として扱う。[1] [2]

また、将来これらを「職場ウイルス度チェック」本体へ統合する場合でも、EP番号、ウイルス名、型の表記は残し、単発企画ではなくシリーズ資産として再利用できる形を維持する。

## 6. 将来展開：職場ウイルス図鑑 年鑑構想

### 位置づけ

「職場ウイルス図鑑 年鑑」は、動画視聴後の**職場あるあるチェック**で集まった回答傾向を、年単位でまとめる将来企画である。現時点では Season1 と Season2 のフォームは参加導線と観測基盤の整備を主目的とするが、将来的にはその蓄積をもとに、シリーズ全体の傾向を可視化する**年鑑**や**白書**へ接続できる構想として位置づける。[1] [2]

### 想定コンテンツ

年鑑化する場合には、単なる集計一覧ではなく、シリーズ世界観と実用導線の両方に接続できる編集単位で整理する。

| 想定コンテンツ | 内容 |
|---|---|
| 年間職場ウイルスランキング | 回答傾向をもとに、その年に反応が集まったウイルスを一覧化する |
| 職場規模別ウイルス傾向 | 職場規模ごとに出やすいウイルスや株の偏りを整理する |
| 季節別・月別の流行傾向 | 年間の中で反応が増えやすい時期を観測レポートとしてまとめる |
| 変異株ランキング | Season2以降で可視化された株の反応傾向を比較する |
| 推奨ワクチン一覧 | Season3で整理した防衛策を、傾向別の対処ヒントとして接続する |
| 職場ウイルス流行語 | 回答や反応が集まった代表セリフやキーワードを年次資産として整理する |
| リーマン自己防衛白書への接続 | 年鑑で見えた傾向を、自己防衛システムや白書企画へつなげる |

### 注意点

将来この構想を実装する場合でも、扱いはあくまで**匿名・集計ベースの職場あるある観測レポート**に留める。個人情報は取得せず、医療診断、心理診断、メンタル診断のような見え方を避けることが前提である。また、Season1の段階ではこの構想を前面に出しすぎず、まずはフォーム回答を蓄積し、十分な件数が集まってから検討する。

| 注意点 | 方針 |
|---|---|
| 個人情報 | 取得しない |
| 統計処理 | 匿名・集計ベースで扱う |
| 表現方針 | 医療・心理診断ではなく、職場あるある観測レポートとして扱う |
| Season1での扱い | 表に出しすぎない |
| 検討タイミング | 回答が十分に蓄積してから判断する |

### 公開レベル方針

集計データは、一度にすべてを公開するのではなく、**コンテンツとして価値が最大化されるタイミング**で段階的に開示する。内部蓄積、簡易公開、体系公開の順で公開レベルを上げることで、視聴者参加の熱量と将来の年鑑価値を両立させる。

| 公開段階 | 方針 | 想定内容 |
|---|---|---|
| 初期 | 非公開 | 内部で回答傾向を蓄積し、設問や集計軸の安定化に使う |
| 中期 | 簡易ランキングのみ公開 | 反応が多いウイルス、株、セリフなどを短いランキング形式で見せる |
| 後期 | 年鑑として体系化して公開 | 年間傾向、職場規模別傾向、変異株ランキング、推奨ワクチン一覧などをまとめて公開する |

この方針により、フォーム回答は単なる裏側データではなく、段階的に価値化される**シリーズ資産**として育てることができる。

## 7. データ処理フロー設計

本企画のフォーム運用は、単に回答を集めるだけでなく、将来的に**年鑑・白書・レポート**へ接続できる観測基盤として設計する。そのため、データ処理は **1. データ収集・蓄積、2. 数値集計、3. 分析・解釈** の三層構造で運用し、役割を混同しないことを原則とする。[1] [2]

### 1. データ収集（フォーム）

回答は Googleフォーム等を用いて収集し、各回答には**EP番号・ウイルス名・型**を必ず紐づける。これにより、どの話にどのような反応が集まったかを後から追跡できるようにする。なお、取得するのは企画運用に必要な最小限の項目に限り、個人情報は取得しない。

| 項目 | 方針 |
|---|---|
| 回答取得手段 | Googleフォーム等を利用する |
| 必須ひも付け | EP番号・ウイルス名・型 |
| 個人情報 | 取得しない |
| 表現方針 | 医療・心理診断ではなく、職場あるあるチェックとして扱う |

### 2. データ集計（スプレッドシート）

フォーム回答はスプレッドシートに蓄積し、**数値の整理**に限定して扱う。ここでは解釈や意味づけを行わず、あくまで後段の分析に渡すための整理層として設計する。

| 項目 | 内容 |
|---|---|
| EP別回答数 | 各ウイルスの反応量を確認する |
| 点数分布 | 潜伏・注意・反応ありの割合を整理する |
| 職場規模別傾向 | Season2以降で使用する |
| 時系列変化 | 日次・週次・月次で変化を見る |

※ この段階では数値の整理のみを行い、解釈は行わない。

### 3. 分析・解釈（NotebookLM）

NotebookLM は、集計済みデータをもとに**言語化・要約・レポート化**を行う分析層として使う。計算そのものを担わせるのではなく、整理済みの数値から傾向を読み取り、企画・レポート・年鑑原稿へ接続するための解釈に用いる。

用途としては、今週の職場ウイルス傾向まとめ、反応が高いウイルスの分析、特徴的なパターン抽出、年鑑・白書用原稿の生成などを想定する。

### NotebookLMへの入力データ

NotebookLM には、次のような資料をまとめて投入する。

| 入力データ | 内容 |
|---|---|
| 集計済みスプレッドシート | CSV 等で整理した集計結果 |
| 各EPの設定情報 | ウイルス名・型・株などの基礎設定 |
| 過去の分析メモ | 以前の観測結果や考察メモ |

### NotebookLMへの指示例

NotebookLM は、たとえば次のような問いで活用する。

| 目的 | 指示例 |
|---|---|
| 週次要約 | 今週の職場ウイルス傾向を要約してください |
| 高反応分析 | 最も反応が高いウイルスは何か |
| パターン抽出 | 特徴的な傾向を3つ挙げてください |
| 比較分析 | 職場規模別の違いを分析してください |
| 原稿化 | 年鑑用の文章としてまとめてください |

### 役割分担の明確化

収集、集計、分析はそれぞれ役割を分けて運用する。

| 役割 | ツール | 内容 |
|---|---|---|
| 収集 | フォーム | 回答を集める |
| 集計 | スプレッドシート | 数値を整理する |
| 分析 | NotebookLM | 意味づけ・文章化する |

### 重要な考え方

このフローでは、**集計と解釈を分離すること**が重要である。NotebookLM は「計算」のためではなく、「解釈」のために使う。また、データは集まった瞬間にすべて公開するのではなく、既存の公開レベル方針に従って段階的に活用する。

| 観点 | 方針 |
|---|---|
| 集計と解釈 | 分離する |
| NotebookLMの役割 | 計算ではなく解釈に使う |
| 公開方針 | データは段階的に活用し、即時に全面公開しない |
| 分析表現 | 断定ではなく「傾向」として扱う |
