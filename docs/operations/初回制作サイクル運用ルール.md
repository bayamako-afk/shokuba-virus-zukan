# 職場ウイルス図鑑｜Season 1 初回制作サイクル運用ルール

## 1. 基本方針
初回制作では、完成度を過剰に追い込むことよりも、**Season 1 を量産できる制作手順を固定すること**を優先する。今回の長期設計では、Season 1 は **違和感の発見編 / 図鑑登録編** として定義されているため、ここで目指すべきなのは反撃や解決ではなく、**職場異変を観測し、名前を与え、一本の記録として成立させること**である。

この前提では、ChatGPT は話の核を深め、ChatGPT image2 は人物の基準を作り、Zopia は最終的なショート映像を作る。一方、**Manusはその間に入り、構造整理、shotlist作成、asset list整理、Season整合確認、図鑑登録運用を担う**。初回サイクルの価値は、1本の完成度だけでなく、**次の10本を回せるフォーマットを残すこと**にある。

---

## 2. Season 1 としての初回サイクルの目的
Season 1 は「退治の物語」ではなく、「認識の物語」である。したがって、初回サイクルでは、視聴者に「その違和感には名前がある」と伝えられる構造を安定して作れるようになることが重要である。

| 項目 | 初回サイクルで達成すべきこと |
|---|---|
| 話の役割 | 違和感の観測と命名 |
| 主人公の役割 | 観測者として引っかかること |
| ラスト | 命名、観測ログ、図鑑登録 |
| 世界観提示 | AI相棒や自己防衛隊は匂わせ程度 |
| 避けること | 解決しすぎ、説明しすぎ、バトル化 |

ここで重要なのは、**Season 2 的な防御やSeason 3 的な連帯を先取りしすぎないこと**である。まずはSeason 1 で世界観を浸透させる必要がある。

---

## 3. 初回サイクルの標準順序
| 順序 | 作業 | 主な担当 | 成果物 |
|---|---|---|---|
| 1 | ウイルス選定 | 企画者 + ChatGPT | エピソードの核 |
| 2 | Season位置づけ確認 | Manus | Season 1 であることの確認メモ |
| 3 | 主人公設計 | 企画者 + ChatGPT | episode sheet 初稿 |
| 4 | 構造整理 | Manus | Season 1 用 episode sheet 整理版 |
| 5 | キャラ基準画像設計 | 企画者 + Manus | character registry |
| 6 | 基準画像生成 | ChatGPT image2 | 基準顔と必要差分 |
| 7 | 参照画像承認 | 企画者 + Manus | approved reference set |
| 8 | カット割り確定 | Manus | shotlist |
| 9 | 映像投入文整形 | Manus | zopia prompt |
| 10 | 図鑑登録演出整理 | Manus | virus registry / ending format |
| 11 | 映像生成と微調整 | Zopia | 縦型ショート初稿 |
| 12 | 公開後レビュー | 企画者 + Manus | release review |

この順序で最も大事なのは、**ManusがSeason 1 の枠から逸脱していないかを途中で確認すること**である。違和感を描く話なのに、途中で解決や反撃が入りすぎるとSeason 1 の量産フォーマットが崩れる。

---

## 4. 守谷守の扱い方
守谷守は固定絶対主人公ではなく、**代表観測者**として扱う。この原則は初回サイクルでも変わらない。初回の主人公が守谷でもよいし、別の観測者でもよいが、どちらの場合も作品は「図鑑の一記録」として成立していなければならない。

| 主人公パターン | 初回での考え方 |
|---|---|
| 守谷守回 | シリーズの視点軸を示しやすい |
| 他観測者回 | オムニバス性と量産性を示しやすい |
| 守谷不在回 | 図鑑世界の広がりを先に感じさせられる |

Manusは、どの主人公パターンを採っても、**観測記録としての統一感**が崩れないよう整理する必要がある。

---

## 5. 自己防衛隊とAI相棒の出し方
Season 1 では、AI相棒や自己防衛隊は**全面展開しない**。存在を感じさせる程度に留めることで、世界観の余白を残す。ここを出しすぎると、Season 2 の進化余地を先に使ってしまう。

| 要素 | Season 1 での扱い |
|---|---|
| AI相棒 | 観測ログ表示、匂わせ、短い表示程度 |
| 自己防衛隊 | 存在の示唆のみ |
| 防御行動 | 明確な対処まで進めすぎない |
| 世界観説明 | 説明しすぎない |

この抑制もまた、Manusが毎話確認すべきポイントである。

---

## 6. 基本構造テンプレート
Season 1 のショートは、できるだけフォーマット化する方が量産に向く。基本構造は、**普通の職場 → 違和感発生 → 観測者が巻き込まれる → 周囲は平然 → 観測者だけが引っかかる → 命名** である。

| 段階 | 役割 |
|---|---|
| 1. 普通の職場風景 | 異常の前提となる日常を見せる |
| 2. じわじわ違和感発生 | 空気の歪みを作る |
| 3. 観測者が巻き込まれる | 当事者性を発生させる |
| 4. 周囲は平然 | 異常が異常扱いされない怖さを出す |
| 5. 観測者だけが引っかかる | 認識の物語にする |
| 6. 命名・図鑑登録 | シリーズ性を成立させる |

Manusは、各話がこの骨格から逸脱していないかを見ながら、15秒 / 30秒 / 45秒に圧縮・展開できるよう整理する。

---

## 7. ラスト演出の統一ルール
Season 1 の終わり方は、作品のブランドに直結する。したがって、**命名・観測ログ・図鑑登録**の演出を共通資産として固定する方がよい。

| 演出要素 | 役割 |
|---|---|
| 画面暗転 | 日常から記録空間への切替 |
| テロップ | ウイルス名の確定 |
| AI観測ログ | 図鑑らしさと世界観の匂わせ |
| 図鑑登録表示 | シリーズ資産としての積み上げ感 |

この共通ルールをManusが管理することで、毎話の終わり方にシリーズ感が生まれる。

---

## 8. 初回で優先すべき代表ウイルス
初回では、視覚化しやすく、短尺で成立しやすく、共感されやすいウイルスを優先するべきである。

| 優先候補 | 初回向きな理由 |
|---|---|
| ため息ウイルス | 空気の変化を一瞬で見せやすい |
| 差し戻し確認ウイルス | 反復と圧迫を短尺にしやすい |
| 丸投げウイルス | コメント共感が起きやすい |
| 口頭指示ウイルス | 記録不在の怖さを出せる |
| 無言差し戻しウイルス | 静かな不気味さが強い |

初回はとくに、**「説明が要らず、空気で伝わるもの」**を優先するべきである。

---

## 9. 生成しないための初回ルール
Season 1 の量産を安定させるには、作れるものを全部作らないことが重要である。人物画像は、**基準顔1、必要表情2〜3、必要なら帰宅後1**を上限の目安にする。背景や小道具も、既存資産で代替できるものは新規生成しない。

| 対象 | 初回ルール | Manusの確認点 |
|---|---|---|
| 基準顔 | まず1案を基準に比較し、必要時のみ追加 | 人物像がまだ曖昧ではないか |
| 表情差分 | カットで必要なものだけ作る | 保険で増やそうとしていないか |
| 衣装差分 | 用途差が明確な時のみ作る | 類似差分で代替できないか |
| 背景 | 流用できるなら新規生成しない | 共通資産で足りないか |
| 動画 | shotlist と参照素材が固まるまで回さない | 構成の曖昧さを生成で埋めていないか |

このルールを毎話適用することで、Season 1 の量産性が上がる。

---

## 10. 初回で必ず残すべき記録
初回制作では、完成動画よりも、**次回再利用できる記録**を残すことが重要である。とくにSeason 1 では、図鑑登録の積み上げがそのまま世界観資産になる。

| 記録対象 | 保存形式 | 主な整理担当 |
|---|---|---|
| 採用したエピソード核 | episode sheet | Manus |
| Season位置づけの根拠 | manuscript note | Manus |
| 採用した image2 用プロンプト | character_base_prompts | Manus |
| 採用した参照画像 | approved reference set | 企画者 + Manus |
| 命名結果 | virus registry | Manus |
| Zopia投入文 | zopia prompt | Manus |
| 公開後の所感 | release review | 企画者 + Manus |

この記録が残っていれば、第2話以降の立ち上がりが大幅に速くなる。

---

## 11. 初回におすすめの実務着手
初回の実務として最も良いのは、**ため息ウイルス**のように視覚化しやすく、空気の伝播を短尺で見せやすい題材を選ぶことである。そこに、守谷守または別観測者を主人公として設定し、最後に命名と図鑑登録で締める形を作れば、Season 1 の基本形を早く検証できる。

次の工程としては、**Season 1 用 episode sheet**、**virus registry 初稿**、**主人公の character registry 記入版**、**ChatGPT image2 用の基準画像プロンプト実作成版**、**Manus による shotlist と asset list**、**Zopia 用ショット指示文**を一気に揃えるのが最も効率的である。
