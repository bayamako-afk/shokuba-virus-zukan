# 職場ウイルス図鑑｜Season 1〜Season 3 シリーズ構成標準

## 1. この文書の目的

本書は、『職場ウイルス図鑑』を**単発の職場あるある短編ではなく、継続視聴されるシリーズ作品として運用するための標準構成**を定義する。今後の制作では、各話を単独で成立させるだけでなく、**Season 1 から Season 3 へ段階的に世界観、緊張感、主人公の防衛意識が進化する設計**を前提にする。

このシリーズの最重要コンセプトは、単なる共感ネタではなく、**「サラリーマン自己防衛システムの物語」**である点にある。したがって、ウイルス現象の発見、分析、防衛、臨戦態勢への移行は、すべて同一世界の連続した成長軸として扱う。

## 2. シーズン全体像

Season 1〜Season 3 は、同じ世界と同じ主人公を維持したまま、**観測者 → 防衛者 → 備える側のリーダー**へ進む三段階構造として設計する。変わるのは緊張の段階と対処の水準であり、世界観の根本をファンタジーへ飛ばすことではない。

| Season | フェーズ名 | 中核コンセプト | 主人公の状態 | 作品としての見え方 |
|---|---|---|---|---|
| **Season 1** | 予兆 | 職場の違和感を観測し、命名する段階 | 気づく人 | リアル寄り、余韻重視、解決しない |
| **Season 2** | 防衛態勢 | 構造を分析し、小規模な防御を始める段階 | 守る人 | 現実寄りの軽SF、ログ・UI・AI補助が増える |
| **Season 3** | 臨戦態勢 | 個人問題を超え、組織構造に備える段階 | 備える人 / 守る側のリーダー | 緊張感拡大、組織視点、複数感染の処理 |

## 3. シーズン別の設計基準

### 3-1. Season 1

Season 1 は、**「何かがおかしい」と気づくためのシーズン**である。ここでは主人公がまだ防御や反撃に入らず、違和感を観測し、視聴者にも「これ、うちの会社にもある」と思わせることを主目的にする。結末は解決ではなく、日常の延長に残る不穏さで止める。

| 項目 | 標準方針 |
|---|---|
| 物語の役割 | 違和感の発見と命名 |
| 主人公の行動 | 防御しない、反撃しない、まず観測する |
| 世界観 | 現実の会社に限りなく近い |
| 演出 | 空気、沈黙、間、周囲の平然さを優先 |
| 禁止寄り要素 | 派手な解決、強いSF化、露骨な戦闘表現 |
| 締めの方向 | 「違和感は正体不明だが確かに存在する」で終える |

Season 1 の締め文は、シリーズの初期認知を揃えるため、疑念と不穏さが残る型に寄せる。命名テロップや断定文を使ってもよいが、余韻の文脈としては**「これは、ただの違和感ではなかった」**、**「何かがおかしい…」**、**「気のせい…なのか？」**といったトーンを標準にする。

### 3-2. Season 2

Season 2 は、**主人公が自宅で分析を始め、職場ウイルスには構造があると認識するシーズン**である。職場だけで完結せず、自宅のAI司令室が登場し、そこで観測記録が整理されることで、シリーズは「経験の蓄積」を持ち始める。ただし、まだ世界観はリアルベースであり、SF演出は現実の延長に見える範囲へ留める。

| 項目 | 標準方針 |
|---|---|
| 物語の役割 | 分析、防衛仮説の導入、小規模な防御成功 |
| 主人公の行動 | 記録する、分析する、防ぐ、完全勝利はしない |
| 世界観 | 現実の会社 + 自宅AI司令室 |
| 演出 | デジタルUI、ログ表示、静かなSF感 |
| 防衛要素 | 議事録バリア、証跡ログシールド、CC透明化ウォール、期限明文化フィールド、口頭指示封じプロトコル |
| 締めの方向 | 「防げるが、まだ完全ではない」で終える |

Season 2 では、1話につき1つの防衛モジュールを導入する。防衛は成功しても限定的であり、**「防げる…だが、完全ではない」**、**「これは一部に過ぎない」**という終わり方で、Season 3 への拡張余地を残す。

### 3-3. Season 3

Season 3 は、**個人の防衛から組織的対峙へ進むシーズン**である。ここでは自己防衛隊の存在が示唆・拡張され、主人公以外の観測者や被感染者も明確に可視化される。複数ウイルスが同時発生し、部署単位ではなく会社全体へ構造が広がることで、シリーズは本格的な「対処の物語」へ入る。

| 項目 | 標準方針 |
|---|---|
| 物語の役割 | 組織構造の可視化、感染拡大への備え、複合事象への対処 |
| 主人公の行動 | 他者を守る、備える、組織視点で判断する |
| 世界観 | 現実の会社 + 拡張された防衛視点 |
| 演出 | 緊張感の増加、スケール拡大、一部アクション的表現は可 |
| 拡張要素 | 感染源マップ、理不尽チェーン解析、同調圧力クラスター、管理職ウイルス変異株、社内パンデミック |
| 締めの方向 | 「戦いは始まった」で終える |

Season 3 の締めは、単なる不気味さではなく、**後戻りできない段階に入った感覚**を残す。標準トーンは **「これは戦いだ」**、**「もう戻れない」** である。

## 4. 主人公の成長軸

主人公である**守屋守**は、全シーズンを通じて同一人物でなければならない。外見、服装系統、雰囲気、観測者としての基礎姿勢は固定し、シーズンを跨いでも連続性が崩れないようにする。そのうえで、各シーズンで役割だけが進化していく。

| 段階 | 主人公の役割 | 必ず残す核 |
|---|---|---|
| **Season 1** | 気づく人 | 現場寄り、真面目、違和感を飲み込みつつ観測する |
| **Season 2** | 守る人 | 観測者でありながら、記録と分析を開始する |
| **Season 3** | 備える人 / 守る側のリーダー | 自分だけでなく他者の防衛まで視野に入れる |

## 5. 制作共通方針

シリーズ全体で守るべき制作方針は、**ショート動画最適化、キャラクター維持、世界観の統一**の三つである。各シーズンの色は変わっても、動画の設計思想は崩さない。

| 共通項目 | 標準方針 |
|---|---|
| ショート動画設計 | 1話 = 1ウイルスを基本とし、完全解決させず、次が気になる連鎖型にする |
| キャラクター維持 | 守屋守は全シーズン共通の同一人物とし、外見・服装・雰囲気を固定する |
| 世界観 | 現実の会社 + わずかな異常を基本とし、完全ファンタジーへは行かない |
| シリーズ性 | 各話は単発消費ではなく、世界観と防衛システムの進化の一部として配置する |

## 6. 今後の運用ルール

今後の全話制作では、**必ずSeasonを先に確定し、そのSeasonの標準テンプレートを使って話を設計する**。Season不明のままエピソードシートを書き始めることは避け、まず「この話は観測段階なのか、防衛段階なのか、臨戦段階なのか」を定めたうえで、演出密度、台詞量、UI強度、締めの重さを決める。

また、既存のSeason 1制作資産は破棄せず、**Season 1標準の厳密化**として位置付ける。Season 2 と Season 3 は、その延長上で新たに標準化し、全体として一つの連続世界を保つ。
