# 職場ウイルス図鑑｜Season 1〜Season 4 シリーズ構成標準

## 1. この文書の目的

本書は、『職場ウイルス図鑑』を**単発の職場あるある短編ではなく、段階的に深まるシリーズ作品として運用するための標準構成**を定義する。今回の改訂では、各Seasonで視聴者へ何を見せるかを制御する**段階公開方式**を導入し、Season 1 での直感的共感、Season 2 での深掘り、Season 3 での実用化、Season 4 でのIP完成を一続きの成長線として整理した。[1] [2]

このシリーズの最重要コンセプトは、単なる共感ネタではなく、**「サラリーマン自己防衛システムへ接続する観測と防衛の物語」**である点にある。したがって、ウイルス現象の発見、変異理解、防衛策の導入、流行構造の可視化は、すべて同一世界の連続した拡張として扱う。

## 2. シーズン全体像

Season 1〜Season 4 は、同じ世界と同じ主人公を維持したまま、**気づく人 → 見分ける人 → 守る人 → 世界を説明できる人**へ進む四段階構造として設計する。変化するのは見せる情報量と対処の深度であり、視聴者に開示する階層を順番に増やしていくことが核である。[1] [2]

| Season | フェーズ名 | 中核コンセプト | 主人公の状態 | 作品としての見え方 |
|---|---|---|---|---|
| **Season 1** | 発見編 | 違和感を観測し、名前を与える段階 | 気づく人 | リアル寄り、説明を削り、共感優先 |
| **Season 2** | 変異編 | 同じウイルスにも複数パターンがあると知る段階 | 見分ける人 | コメント考察が生まれる、分類が深まる |
| **Season 3** | 防衛編 | 防衛策を言語化し、実用性へ接続する段階 | 守る人 | リーマン自己防衛システムへの接続が見える |
| **Season 4** | 流行編 | 感染構造と上司分布まで可視化し、世界観を完成させる段階 | 世界を説明できる人 | 図鑑IP、資料、書籍、ビジネス展開へ接続 |

## 3. 段階公開方式の標準

段階公開方式では、設定そのものを後から作るのではなく、**最初から裏設定として保持し、Seasonごとに見せるレイヤーだけを切り替える**。この方針により、Season 1 の軽さを失わずに、後続Seasonで自然な拡張ができる。[1]

| 要素 | Season1 | Season2 | Season3 | Season4 |
|---|---|---|---|---|
| ウイルス名 | 表示 | 表示 | 表示 | 表示 |
| 型 | 表示 | 表示 | 表示 | 表示 |
| 株 | 非表示 | 表示 | 表示 | 表示 |
| ワクチン | 最小限 | 最小限 | 表示 | 表示 |
| 感染レベル | 非表示 | 非表示 | 裏設定 | 表示 |
| 上司分布 | 非表示 | 非表示 | 非表示 | 表示 |

この設計により、Season 1 は**理解しやすい入口**、Season 2 は**議論したくなる深さ**、Season 3 は**役に立つ接続点**、Season 4 は**世界観が完成した図鑑体験**として機能する。

## 4. シーズン別の設計基準

### 4-1. Season 1

Season 1 は、**「何かがおかしい」と気づくためのSeason**である。ここでは主人公がまだ防御や反撃に入らず、視聴者へも説明を与えすぎない。目的は、図鑑の完成度よりも、職場あるあるとしての直感的共感を最大化することである。[1]

| 項目 | 標準方針 |
|---|---|
| 物語の役割 | 違和感の発見と命名 |
| 主人公の行動 | 防御しない、反撃しない、まず観測する |
| 主要表示情報 | ウイルス名、型 |
| 非表示情報 | 株、感染レベル、上司分布 |
| ワクチンの扱い | 最小限または非表示 |
| 世界観 | 現実の会社に限りなく近い |
| 締めの方向 | 未解決のまま違和感を残す |

Season 1 の終わり方は、**「気づき」で止めること**が最重要である。命名はしても解決しない。視聴者には「これ、うちにもある」と思わせ、守屋守には「これは気のせいではない」と気づかせる段階に留める。

### 4-2. Season 2

Season 2 は、**同じウイルスにも変異パターンがある**と見せるSeasonである。ここでは株を前面化し、視聴者が「これは○○株だ」と語りたくなる構造を意図的につくる。深掘りはするが、防衛や完全対抗を前に出しすぎないことが重要である。[1] [2]

| 項目 | 標準方針 |
|---|---|
| 物語の役割 | 変異理解、分類の深化、視聴者考察の誘発 |
| 主人公の行動 | 違いを見分ける、記録する、比較する |
| 主要表示情報 | ウイルス名、型、株 |
| ワクチンの扱い | 最小限。裏に存在するが主役にしない |
| 世界観 | 現実ベースのまま分類精度が上がる |
| 締めの方向 | これは一種ではない、と気づく |

### 4-3. Season 3

Season 3 は、**防衛策を実用的な言葉として提示するSeason**である。ここで初めてワクチンを前面化し、リーマン自己防衛システムとの接続点を明示する。ただし、万能な解決ではなく、あくまで使える整理道具や防衛モジュールとして見せる。[1] [2]

| 項目 | 標準方針 |
|---|---|
| 物語の役割 | 防衛策の導入、実用性の付与、自己防衛システムへの接続 |
| 主人公の行動 | 守る、記録を活かす、限定的に防ぐ |
| 主要表示情報 | ウイルス名、型、株、ワクチン |
| 感染レベルの扱い | 裏設定として運用し、必要時も前面化しない |
| 世界観 | 現実寄りのまま実用モジュールが増える |
| 締めの方向 | 防げるが、まだ完全ではない |

### 4-4. Season 4

Season 4 は、**感染レベルと上司分布まで可視化し、図鑑世界を完成させるSeason**である。ここでは個別のあるあるネタを超え、組織構造、流行傾向、上司タイプ別分布まで見せることで、シリーズをIP、資料、書籍、研修、ビジネス展開へ接続できる状態にする。[1] [2]

| 項目 | 標準方針 |
|---|---|
| 物語の役割 | 流行構造の可視化、図鑑世界の完成、IP化への接続 |
| 主人公の行動 | 個別事象を超えて全体構造を説明する |
| 主要表示情報 | ウイルス名、型、株、ワクチン、感染レベル、上司分布 |
| 世界観 | 図鑑・分析・資料化まで耐えうる整理度 |
| 展開先 | 書籍、資料、診断、ビジネス提案、IP拡張 |
| 締めの方向 | これは個人の違和感ではなく、職場全体の流行構造である |

## 5. 主人公の成長軸

主人公である**守屋守**は、全Seasonを通じて同一人物でなければならない。外見、服装系統、雰囲気、観測者としての基礎姿勢は固定し、Seasonを跨いでも連続性が崩れないようにする。そのうえで、各Seasonで役割だけが進化する。[1]

| 段階 | 主人公の役割 | 必ず残す核 |
|---|---|---|
| **Season 1** | 気づく人 | 現場寄り、真面目、違和感を飲み込みつつ観測する |
| **Season 2** | 見分ける人 | 違和感を記録し、違いに名前を与え始める |
| **Season 3** | 守る人 | 観測者でありながら、自分や周囲を守る手段を持つ |
| **Season 4** | 世界を説明できる人 | 個別体験を構造へ変換し、他者へ共有できる |

## 6. 制作共通方針

シリーズ全体で守るべき制作方針は、**ショート動画最適化、キャラクター維持、世界観の統一、情報開示の順序制御**の四つである。各Seasonで見せるものが増えても、入口のわかりやすさを失ってはならない。[2]

| 共通項目 | 標準方針 |
|---|---|
| ショート動画設計 | 1話 = 1ウイルスを基本とし、連鎖視聴を促す |
| キャラクター維持 | 守屋守は全Season共通の同一人物とし、外見・服装・雰囲気を固定する |
| 世界観 | 現実の会社を基盤にしつつ、図鑑としての整理度を段階的に上げる |
| 情報開示 | 裏設定は保持し、Seasonごとに見せる要素のみを切り替える |
| Season 1 の原則 | 説明しない勇気を優先し、共感の速度を落とさない |

## 7. 今後の運用ルール

今後の全話制作では、**必ずSeasonを先に確定し、そのSeasonで表示してよい情報レイヤーを先に決める**。Season不明のままエピソードシートや台本を書き始めることは避け、まず「この話は何を見せるSeasonなのか」を定めたうえで、表示要素、台詞量、演出密度、締め方を設計する。[2]

また、既存のSeason 1 制作資産は破棄せず、**軽量表示へ最適化した発見編の基準**として位置づける。株、ワクチン、感染レベル、上司分布は削除せず、シリーズ資産として残し続けることを原則とする。
