# Season1 メディアミックス管理表

## 1. 文書の目的

本書は、『職場ウイルス図鑑 Season1』の EP001〜EP005 について、ショート動画、漫画、図鑑、Kindle単話版、診断フォーム、SNS投稿、まとめ版再利用候補の進行状況を**1枚で確認できる管理表**として運用するための文書である。Season 1 は量産と観測が優先であるため、各話を個別最適で管理するのではなく、横断的に抜け漏れを確認できる状態をつくることが重要である。[1] [2]

同時に、本管理表は、各媒体を固定的な主従関係で扱うのではなく、**『職場ウイルス図鑑』という世界観を広げる感染経路の束**として管理するための観測台帳でもある。したがって、媒体単体の進捗だけでなく、媒体間の回遊導線が機能しているかも確認対象に含める。

また、初期フェーズでは**公式サイトを必須にしないサイトレス・パンデミック戦略**を採用する。そのため、表向きには発信源を一本化せず、自然発生的に各地で観測される印象を優先する。一方で、裏側では本管理表を通じて、EP一覧、投稿状況、Kindle単話版、診断フォームURL、SNS導線、簡易受け皿URLを一元管理する。

同時に、本管理表は、出版を後工程ではなく**初期展開チャネル**として扱うための運用表でもある。従来のように「Kindle」一列でまとめるのではなく、**Kindle単話版（感染速報版）**と**まとめ版再利用候補**を分けて管理することで、入口商品の進行と後続商品の仕込みを混同せずに追えるようにする。[1]

さらに、Kindle単話版は将来的な英語版並行展開を見据え、日本語版の画像や8コマ構成を流用しながら、英語タイトル、英語セリフ、英語図鑑ページへ差し替えやすいかも備考欄で追えるようにする。これにより、日本語版制作の時点で、英語版への横展開可能性も同時に観測できる。

## 2. ステータス表記ルール

本管理表では、各制作物の進行状況を簡潔に比較できるよう、まずは**未 / 済**の二段階で管理する。必要に応じて、運用上は「着手中」「保留」などを追記してもよいが、標準表示は二段階を基本とする。[1]

| 表記 | 意味 |
|---|---|
| 未 | まだ未着手、または公開・完成前 |
| 済 | 作成済み、または公開済み |

## 4. 運用メモ

この管理表は、単なる完了チェックではなく、**Season 2 で何を優先可視化するかを見極めるための観測台帳**としても使う。したがって、更新時には完成有無だけでなく、反応の強さ、導線の機能、再編集候補も備考欄へ残すことが望ましい。[1] [2]

特にKindle単話版は、売上回収のためではなく、認知拡大、シリーズ一覧性、動画や診断フォームへの送客のための入口商品として扱う。そのため、単話版の有無だけでなく、**まとめ版へ再利用しやすいか**という観点も同時に追う必要がある。[1]

また、英語版並行展開を見据える場合、日本語版の画像、8コマ構成、図鑑レイアウトを共通資産として維持し、テキスト差し替えだけで横展開できるかも観測対象になる。したがって、素材管理上の共通化度合いも、今後は重要な運用指標とする。

さらに、職場ウイルス・パンデミック戦略では、各媒体の単独成果だけでなく、**どの感染経路が次の媒体へ最も強く送客したか**を追う必要がある。したがって、今後は Kindle→動画、動画→漫画、漫画→診断フォーム、診断フォーム→SNS、SNS→各媒体という回遊の強さも、主要観測項目に含める。

加えて、サイトレス運用では、公式サイトなしでも十分に回遊が成立するかを見極める必要がある。そのため、Linktree、note固定記事、Kindle著者ページ、各SNSプロフィールなどの**簡易受け皿**が、どの程度補助導線として機能したかも観測する。

| 観測観点 | 見るポイント |
|---|---|
| 動画 | 再生数、視聴維持率、コメント量 |
| 漫画 | SNS保存率、反応コメント、再掲しやすさ |
| 図鑑 | 整理価値、流用しやすさ |
| Kindle単話版 | 入口商品としての魅力、巻末導線の機能、一覧性 |
| まとめ版再利用候補 | 5話版・10話版へ組み込みやすいか |
| 英語版転用性 | 共通画像の流用可否、英語差し替えのしやすさ |
| 診断フォーム | 回答数、結果分布、話題化のしやすさ |
| SNS投稿 | クリック率、コメント誘導の強さ |
| 感染経路の強さ | 媒体間の送客率、回遊率、再接触率 |
| 簡易受け皿の機能 | Linktree、note、著者ページ、プロフィール導線の補助力 |
