# 職場ウイルス図鑑｜継続制作のための運用設計書

## 1. 文書の目的
本書は、**『職場ウイルス図鑑』を継続的に量産・検証・昇格できる長期シリーズとして運用するための設計書**である。今回の改訂では、Season 1 から Season 4 以降までの役割を明確に切り分け、シリーズ全体を **観測 → 防衛 → 連帯 → 構造分析** へ進化させる設計へ更新した。

本作の前提は、企画の中核をChatGPTで深め、キャラクターベース画像は**ChatGPT image2**で固定し、GitHubで正本管理を行い、NotebookLMとGeminiは構造整理と横断参照に用い、Zopiaは最終的な映像化に限定して使う、というものである。ただし、この体制においても**Manusの役割は消えない**。むしろツールが増えるほど、何をどの順番で確定し、何を生成せずに済ませ、どこでシリーズ整合を保つかを制御する担当が必要になる。その制作管制・前処理・整合管理の中心がManusである。

---

## 2. 作品全体の核
『職場ウイルス図鑑』は、単なる職場あるある集でも、毎回スカッと解決するドラマでもない。本作の核は、**日本の職場に潜む見えない理不尽を“職場ウイルス”として観測し、命名し、記録していくこと**にある。つまり、まず描くべきなのは退治ではなく認識である。

この前提に立つと、シリーズ全体は「違和感を見つける物語」として始まり、そこから少しずつ自己防衛の技術、観測者同士のつながり、そして構造そのものへの視点へ進んでいく。したがって、本シリーズの魅力は一話完結のフックだけではなく、**ショート動画の量産性と長期的な世界観進化を同時に持てること**にある。

| 観点 | 本シリーズの定義 |
|---|---|
| ジャンル感 | ブラックコメディ寄りの職場異変観測ドラマ |
| 基本体験 | 「うちにもある」と気づいた瞬間に名前が与えられる |
| 初期の主目的 | 違和感を可視化し、図鑑として登録する |
| 長期の主目的 | 個人の違和感を、自己防衛・連帯・構造理解へ育てる |
| 作品の後味 | モヤモヤ、不気味さ、納得、静かな覚醒 |

---

## 3. 守谷守の位置づけ
今回の改訂でもっとも重要な変更点の一つは、**守谷守を固定絶対主人公ではなく、代表観測者として定義すること**である。守谷守は作者自身の実感や視点を色濃く反映した投影的キャラクターであり、世界観の中核となる視点人物ではあるが、毎話必ず彼だけを主人公にする必要はない。

この再定義によって、作者性を失わずに量産性と拡張性を両立できる。各話の主人公は守谷でもよいし、別の観測者、被感染者、関係者でもよい。守谷が直接登場しない回も、**職場ウイルス図鑑という世界の観測記録**として成立する。シリーズ全体は、守谷一人の物語ではなく、**複数の観測事例が蓄積される記録集**として設計される。

| 要素 | 定義 |
|---|---|
| 守谷守 | 代表観測者、作者性の軸、世界観の中核視点 |
| 各話の主人公 | 守谷または他の観測者、被感染者、関係者 |
| シリーズ全体 | 職場ウイルスの観測記録集 |
| 守谷の役割変化 | Season が進むほど自己防衛隊や構造理解との接続が深まる |

この構造にしておくことで、**守谷中心回**と**オムニバス回**を無理なく共存させられる。ここは量産と長期シリーズ化の両方に効く重要な設計である。

---

## 4. Season 1〜4 の長期アーキテクチャ
本シリーズは、最初から毎回「反撃・解決」まで描く方式ではない。むしろ、Seasonごとに役割を変えながら少しずつ世界観を拡張していく。その全体像を先に明文化しておくことで、各話の作り方と各シーズンの目的がぶれにくくなる。

| Season | 主軸 | 作品上の役割 | 視聴後感 | 制作目的 |
|---|---|---|---|---|
| Season 1 | 違和感の発見 | 職場異変を観測し、命名し、図鑑登録する | じわ怖・モヤモヤ | 大量制作・浸透 |
| Season 2 | 防御の開始 | 知った者が静かに自己防衛を始める | なるほど・少し救われる | 深化・進化 |
| Season 3 | 観測者の拡張 | 複数観測者のネットワーク化を見せる | 自分も仲間かもしれない | 横展開・寿命延長 |
| Season 4 | 構造の可視化 | 個人ではなく会社構造そのものを描く | 深い納得・社会批評感 | 格上げ・テーマ深化 |

この四段構えにしておくことで、『職場ウイルス図鑑』は単発ネタ集で終わらず、**観測 → 防衛 → 連帯 → 構造分析**へ進化する長期シリーズになる。

---

## 5. Season 1 の定義
Season 1 は、**違和感の発見編 / 図鑑登録編**である。ここで描くべきなのは、観測者が職場の理不尽や異常な空気に違和感を覚えること、しかしその場では明確に反撃・解決しないこと、周囲は平然としており異常が異常として扱われないこと、そして最後にその現象へ**「〇〇ウイルス」**という名前が与えられることである。

Season 1 は退治の物語ではなく、**認識の物語**である。視聴者に対して「その違和感には名前がある」と伝えることが最大の目的であり、世界観説明を過剰に入れる必要はない。むしろ、説明しすぎないことが不気味さと余韻を生む。

| 項目 | Season 1 の基本 |
|---|---|
| テーマ | 違和感の発見と命名 |
| 主人公 | 守谷守または他の観測者 |
| 終わり方 | 命名、図鑑登録、観測ログ化 |
| AI相棒 | 匂わせ程度 |
| 自己防衛隊 | 存在を感じさせる程度 |
| 演出軸 | 一見普通だが何かおかしい |

Season 1 はショート動画として量産しやすい。なぜなら、反撃や解決まで描かなくてよく、「異変の入口」だけで成立し、最後にウイルス名を出すだけで一本が締まるからである。したがって、現時点で最優先すべきは**Season 1 量産フォーマットの整備**である。

---

## 6. Season 2 の定義
Season 2 は、**自己防衛隊起動編 / 防衛OSアップデート編**である。Season 1 で名前を知った職場異変に対して、ここでは主人公側が徐々に対応手段を持ち始める。AI相棒、自己防衛隊、リアルタイム支援、証拠保全、感情応答抑制などが、初めて本格稼働し始める。

ただし、Season 2 をバトルものとして描く必要はない。ここで目指すべきは、**静かな防御戦**である。派手な必殺技よりも、「記録を残す」「即答しない」「確認質問に切り替える」「相手の圧を空振りさせる」といった現実的な自己防衛が映える。

| 項目 | Season 2 の基本 |
|---|---|
| テーマ | 防御・対応の開始 |
| 主人公 | 守谷守または防御側観測者 |
| 終わり方 | 防衛、回避、空振り、次段階への接続 |
| AI相棒 | 本格支援 |
| 自己防衛隊 | 機能が明確化 |
| 演出軸 | 静かな防御戦 |

Season 2 は、Season 1 の人気回や刺さったウイルスを深掘りしやすく、横型長尺化とも相性が良い。したがって、量産の主戦場はSeason 1に置きつつ、**当たり回をSeason 2 的進化へ昇格させる**設計が合理的である。

---

## 7. Season 3 の定義
Season 3 は、**観測者拡張編 / 連帯発生編**である。ここでは守谷守だけでなく、別部署、別会社、別立場の観測者や被感染者、防衛実践者が明確に登場し始める。世界観は個人視点から、**複数観測者のネットワーク**へ拡張される。

Season 3 の価値は、視聴者に「自分だけが気づいていたのではなかった」「この図鑑には他にも観測者がいる」と感じさせる点にある。単発エピソードとして成立しながら、背後にネットワークの気配が見えることが重要である。

| 項目 | Season 3 の基本 |
|---|---|
| テーマ | 観測者の拡張と連帯 |
| 主人公 | 複数観測者、被感染者、防衛実践者 |
| 終わり方 | 接続や連帯の示唆 |
| AI相棒 | 観測者ごとの差異も出せる |
| 自己防衛隊 | 仲間・ネットワーク化 |
| 演出軸 | オムニバス性とつながりの両立 |

この段階まで来ると、シリーズの寿命は大きく伸びる。守谷中心回と他主人公回を柔軟に回しやすくなり、世界観の厚みも増す。

---

## 8. Season 4 の定義
Season 4 は、**構造対決編 / 社会拡張編**である。ここで初めて、個々のウイルス事象を超えて、会社構造・文化・制度・空気そのものへ視点を広げる。敵は個人ではなく、**構造**であることがより明確になる。

このフェーズでは、「会議文化」「評価制度」「責任分散」「曖昧指示」など、感染源の奥にある職場システムそのものを可視化できる。守谷守や他の観測者は、単なる被害者ではなく、**記録者・分析者**として進化し、図鑑は単なる記録から社会的な鏡へ変わっていく。

| 項目 | Season 4 の基本 |
|---|---|
| テーマ | 構造そのものの可視化 |
| 主人公 | 守谷守＋複数観測者 |
| 終わり方 | 構造の露出、社会的余韻 |
| AI相棒 | 分析支援・構造可視化 |
| 自己防衛隊 | 思想や仕組みへ昇華 |
| 演出軸 | 怪談性から寓話性へ拡張 |

これにより、『職場ウイルス図鑑』は「会社員向け怪談」から「会社社会の寓話」へ格上げされる。

---

## 9. Manusの役割
このシリーズにおけるManusの役割は、監督の代行ではない。Manusは、**制作進行・構造整理・前処理・整合管理・資産運用の司令塔**である。特に、ツールが増え、Season設計が階層化された今、Manusの役目はむしろ明確になっている。

| Manusの担当領域 | 具体的な役目 | 効果 |
|---|---|---|
| Season整合管理 | いま作っている回がSeason 1 / 2 / 3 / 4 のどこに属するか整理する | シリーズ進化がぶれない |
| episode前処理 | 話の核、主人公、ウイルス、終わり方をテンプレート化する | 企画の輪郭が安定する |
| shotlist作成 | カット単位の見せ方へ落とす | Zopiaの試行回数を減らす |
| asset list整理 | 背景、小道具、UI、不足参照を洗い出す | 無駄な生成を減らす |
| 重複チェック | 過去話と似すぎていないか確認する | マンネリを防ぐ |
| 図鑑管理 | ウイルス名、登録順、観測ログを整理する | 図鑑としての厚みが出る |
| 公開後レビュー整理 | 視聴反応を次話の判断材料へ変換する | 当たり回を育てやすい |

つまり、**ChatGPTは企画を深め、image2は人物を固定し、Zopiaは映像を作る**。それに対して、**Manusはシリーズとして回る制作ラインを設計し、各話を迷わず進めるための前処理を担う**。ここがManusの不可欠な役割である。

---

## 10. ツールごとの役割分担
ツールの役割が曖昧だと、同じ工程を複数箇所で重複して進めてしまい、管理も実行回数も肥大化する。したがって、『職場ウイルス図鑑』では、各ツールの担当範囲を明確に分ける。

| ツール | 主担当 | 役割の定義 | 原則 |
|---|---|---|---|
| ChatGPT | 企画開発 | 世界観整理、各話アイデア、脚本叩き台、構成 | 発想と設計の中核 |
| ChatGPT image2 | キャラ基準画像 | 主人公や必要人物の基準顔、必要最小限の差分 | キャラ画像の標準ルート |
| GitHub | 正本管理 | 企画書、各話シート、テンプレート、台帳、版管理 | 最終確定版は必ずここ |
| NotebookLM | 横断参照 | 企画書、過去話、レビュー、コメント傾向の参照 | 記憶補助として使う |
| Gemini | 構造補助 | 候補比較、長文整理、分類、差分抽出 | 思考補助に限定 |
| Zopia | 映像生成 | カット単位の映像化、ショート初稿の作成 | 最終工程でのみ使う |
| Manus | 制作管制 | 前処理、テンプレ化、整合確認、図鑑管理、生成抑制判断 | シリーズ運用の司令塔 |

低実行運用では、生成能力そのものよりも、**どの時点で生成を止めるか、どこまで言語で確定するか**の方が重要である。その判断補助を担う点でも、Manusの価値は大きい。

## 11. 推奨する制作順序
ツールを自由に往復するより、**順序を固定して判断ゲートを置く**方が強い。特にSeason 1量産フェーズでは、フォーマットの安定化が最重要である。

| 段階 | 主に使う場所 | 何をするか | 次へ進む条件 |
|---|---|---|---|
| 企画発想 | ChatGPT / Gemini | ウイルス候補、主人公候補、切り方候補を出す | その回の核が一文で言える |
| 構造整理 | Manus | Season位置づけ、episode骨格、終わり方を整理する | episode sheet の骨格ができる |
| 確定 | GitHub | 採用案を版として固定する | episode sheet が埋まる |
| キャラ固定 | ChatGPT image2 | 基準顔と必要最小限の差分を作る | approved reference set ができる |
| 制作前処理 | Manus | shotlist、asset list、参照不足を洗う | Zopia投入準備が整う |
| 映像化 | Zopia | 前編ショートを作る | 検証用初稿が出る |
| 反応整理 | Manus | コメント、維持率、続き需要をレビュー化する | 昇格判断の根拠が残る |

この順序で重要なのは、**Manusが発想と生成の間に入ること**である。これにより、構成の曖昧さを生成で埋める悪循環を避けられる。

## 12. Season 1 の量産ルール
現在の最優先はSeason 1 である。したがって、設計書としてもSeason 1 の量産ルールを明文化しておく必要がある。Season 1 では、違和感の観測と命名を中心に据え、反撃や設定説明をやりすぎないことが重要である。

| 項目 | Season 1 運用ルール |
|---|---|
| 基本尺 | 15秒 / 30秒 / 45秒のショート型を作る |
| 基本構造 | 普通の職場 → 違和感発生 → 観測者だけが引っかかる → 命名 |
| ラスト演出 | 暗転、テロップ、AI観測ログ、図鑑登録表示 |
| 主人公設計 | 守谷守回も他観測者回も許容する |
| 説明量 | 説明しすぎない |
| 世界観提示 | AI相棒と自己防衛隊は匂わせ程度 |
| 目的 | 図鑑登録の積み上げと浸透 |

この運用を守ることで、Season 1 は量産しやすくなり、Season 2 以降へ進めるための土台が形成される。

## 13. 代表ウイルスと図鑑管理
Season 1 を回すためには、まず**ウイルス名リストの棚卸し**が必要である。シリーズ感は、単に似た話を量産することではなく、命名と登録の蓄積から生まれる。

| 優先候補 | 位置づけ |
|---|---|
| ため息ウイルス | 空気の変化を見せやすく初回向き |
| 差し戻し確認ウイルス | 反復構造がショートに向く |
| 丸投げウイルス | 共感コメントを得やすい |
| 口頭指示ウイルス | 記録不在の不安が強い |
| 言った言わないウイルス | 認識のズレを描きやすい |
| 会議長引かせウイルス | 構造性の芽を見せやすい |
| 善意偽装ウイルス | ブラックコメディ性が強い |
| 空気読め圧ウイルス | 集団圧の不気味さを出しやすい |
| 無言差し戻しウイルス | 演出が静かで怖い |
| 責任なすりつけウイルス | 当事者性が強く刺さりやすい |

ここでもManusは、**命名の重複管理、登録番号、似た現象の整理、Season昇格候補の仕分け**を担うべきである。図鑑として積み上がっていく感覚を運用面で支えることが重要だからである。

## 14. 低実行運用の原則
本シリーズでは、生成できる場面でも、あえて生成しない設計が重要である。キャラクターベース画像は image2 を標準にしつつ、**作る前に決め切る**ことを優先する。その制作制御もManusが担う。

| 対象 | 生成してよい条件 | 止めるべき条件 |
|---|---|---|
| 基準顔 | 役割、年齢感、職場文脈が確定している | 人物像がまだ曖昧 |
| 表情差分 | カットで必要な感情が明確 | 念のため多めに作りたいだけ |
| 衣装差分 | 用途差が明確 | 候補を増やしたいだけ |
| 背景 | 既存資産で代替できない | 類似背景を毎回新規生成しようとしている |
| 動画 | shotlist と参照素材が固まっている | 構成を考えながら生成しようとしている |

低実行とは、単に回数を減らすことではない。**迷いを前段で処理し、制作の判断を生成前に終わらせること**である。その意味でも、Manusの前処理機能はシリーズの中核である。

---

## 15. GitHub管理の最小単位
シリーズ運用を安定させるため、1話ごとに最低限必要なファイルも明文化しておく。これらは、制作再開時の復元性だけでなく、Season設計と図鑑運用を支える。

| ファイル | 内容 | 主な担当 |
|---|---|---|
| `episode_sheet.md` | 話の核、Season位置づけ、主人公、終わり方 | 企画者 + Manus |
| `character_registry.md` | 守谷または他観測者の人物定義 | Manus整理、企画者確認 |
| `character_base_prompts.md` | image2 用基準画像プロンプト | 企画者 + Manus |
| `shotlist_vertical.md` | ショート用カット設計 | Manus |
| `zopia_prompt.md` | Zopia投入用指示 | Manus |
| `asset_list.csv` | 背景、小道具、UI、不足素材 | Manus |
| `virus_registry.md` | ウイルス名、登録番号、類型 | Manus |
| `review_note.md` | 公開後レビューとSeason昇格候補 | Manus |

この最小単位が揃っていれば、各話は単なる動画ではなく、**図鑑の一項目**として蓄積される。

---

## 16. 今後の実務優先順位
今後、Manus向けに優先すべき実務は明確である。第一に、**Season 1 の量産フォーマットを整備すること**。第二に、**ウイルス名リストを棚卸しすること**。第三に、**各ウイルスに対して短い発生シーンを設計すること**。第四に、**15秒 / 30秒 / 45秒のショート型を揃えること**。第五に、**反撃ではなく違和感で終える演出を統一すること**である。

この順序で進めることで、Season 1 を土台としてシリーズを浸透させ、その後にSeason 2 以降へ自然に拡張できる。
