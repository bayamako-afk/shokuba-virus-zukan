# 職場ウイルス図鑑 - Data Strategy

本ドキュメントは、「職場ウイルス図鑑」プロジェクトにおけるデータ収集、蓄積、および活用の戦略を定めたものです。

## 1. データ戦略の目的
本プロジェクトにおけるデータ戦略は、単なるコンテンツの反響測定にとどまらず、将来的な「職場ウイルス感染状況」の独自ビッグデータ化、年鑑化、企業向け脆弱性診断といったビジネス展開の基盤を構築することを目的とします。

## 2. データ収集のチャネル
データは、以下の複数のチャネル（感染経路）から多角的に収集します。

*   **Googleフォーム（感染度チェック）**: 1話1フォーム、原則3問構成で、特定の「○○ウイルス」に対する視聴者の自己診断結果（感染度）を収集します。
*   **SNSエンゲージメント**: YouTube Shorts、TikTok、Instagram Reelsにおける再生回数、いいね数、シェア数、コメント内容（共感の度合い、具体的な体験談など）を定性的・定量的に収集します。
*   **Kindle出版（感染速報版）**: ダウンロード数、レビュー内容、ページ読了率などを通じて、コンテンツへの深い関与度を測定します。

## 3. データ公開の段階的アプローチ
収集したデータの公開は、コンテンツとしての価値を最大化するタイミングで、以下の3段階で段階的に行います。

*   **初期フェーズ（内部蓄積）**: 収集したデータは非公開とし、内部での蓄積と分析に専念します。Season 2以降の本格図鑑化や、ウイルスの「株分類」（派生パターンの特定）に向けた仮説構築の材料とします。
*   **中期フェーズ（簡易公開）**: 一定のデータが蓄積され、傾向が見え始めた段階で、「最も感染が広がっているウイルスランキング」や「特定の業界で猛威を振るうウイルス」といった簡易的な集計結果を、SNSコンテンツやKindleの特集記事として公開し、話題喚起を図ります。
*   **後期フェーズ（体系化・事業化）**: 蓄積されたデータを体系化し、「職場ウイルス年鑑」としての出版や、企業の人事・組織開発部門向けの「職場環境の脆弱性診断サービス」など、データそのものを価値ある事業として展開します。

## 4. データ管理のルールと制約
データの管理およびインフラ選定において、以下のルールを遵守します。

*   **Supabaseの不使用**: 本プロジェクト（GUNOプロジェクト以外）において、データベースやバックエンドサービスとしてSupabaseは使用しません。代替となるデータ管理手法（スプレッドシート、他のBaaS、自社サーバー等）を検討し、ユーザーの意向を反映します。
*   **個人情報の保護**: 初期段階では個人を特定できる情報は収集せず、匿名データとして取り扱います。将来的に企業向け診断等で個人情報を取り扱う場合は、適切なプライバシーポリシーとセキュリティ対策を講じます。
