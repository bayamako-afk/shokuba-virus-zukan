# 職場ウイルス図鑑｜Media Mix Strategy

## 1. 文書の位置づけ

本書は、『職場ウイルス図鑑』のメディアミックス戦略を、**「反応が良ければ後から出版する」方式**から、**「最初から出版も同時展開する」方式**へ切り替えたうえで再定義する上位戦略文書である。ここでいう出版は後工程ではなく、YouTube Shorts、TikTok、Instagram Reels、8コマ漫画と並ぶ**初期拡散チャネル**として扱う。

本企画の核は、職場の違和感を「観測対象」として見立て、その観測結果を複数媒体へ同時に流し込みながら、共感、参加、記録、再編集を積み上げていく点にある。したがって、出版もまた完成物の置き場ではなく、**感染拡大チャネル**であり、同時に**観測対象**でもある。

| 項目 | 本書で定義すること |
|---|---|
| 企画の正体 | 観測・分類・物語化・防衛へ接続する社会会社生活IP |
| 戦略の中心 | 個別媒体ではなく『職場ウイルス図鑑』という世界観そのもの |
| 初期拡散チャネル | Shorts、TikTok、Instagram Reels、8コマ漫画、Kindle単話版、診断フォーム |
| 初期受け皿 | 公式サイト必須ではなく、Linktree、note固定記事、Kindle著者ページ、SNSプロフィールを併用 |
| 出版の役割 | 後工程ではなく初期接点、面展開、導線設計 |
| 言語展開 | まず日本語版を基準化し、将来的に英語版へ差し替えやすく設計 |
| 守る前提 | Seasonごとの段階公開、軽量運用、可視化の段階制御 |

## 2. 方針転換の要点

これまでの想定では、SNSやショート動画で反応が良かった話を後からKindle化する構造が中心だった。しかし今後は、この順序を改める。**売れたら出版するのではなく、最初から出版も同時に走らせる**ことを標準にする。

この変更により、各EPは動画、SNS、漫画だけでなく、**Kindle単話版「感染速報版」**としても初期段階から並列に存在する。つまり、出版は結果ではなく、最初から設計に組み込まれた入口商品になる。

| 旧方針 | 新方針 |
|---|---|
| 反応が良かったら後からKindle化 | 最初からKindleも同時展開する |
| 出版は後工程 | 出版は初期拡散チャネル |
| Kindleは再編集の結果物 | Kindleは観測と誘導の入口商品 |
| 出版で売上回収を狙う | 出版で認知拡大と導線形成を狙う |

## 3. 基本コンセプト

『職場ウイルス図鑑』の中核コンセプトは、**「現代の職場に潜む内的ウイルスを観測し、命名し、別メディアへ同時展開しながら育てるIP」**である。ここで重要なのは、1媒体で当ててから他媒体へ横展開することではなく、同じ原作核を複数の入口から同時に差し出すことである。

そのため、各メディアは主従ではなく役割分担で考える。動画は最速で刺し、漫画は物語として定着させ、Kindleは読める形で面展開し、フォームは参加を回収し、後続の年鑑や白書は傾向を蓄積する。

さらに今後は、この設計を**「職場ウイルス・パンデミック戦略」**、または**「感染拡大型メディアミックス戦略」**として明示する。中心に置くのは本、SNS、ショート動画、8コマ漫画、診断フォームのいずれかではなく、あくまで**『職場ウイルス図鑑』という世界観**である。各媒体は主役ではなく、世界観を広げるための**感染経路**として扱う。

加えて初期フェーズでは、これを**「サイトレス・パンデミック戦略」**として運用する。つまり、公式サイトを最初から必須にせず、**どこから出てきたかわからないが各地で観測される職場ウイルス**という自然発生感を優先する。その一方で、裏側ではEP一覧、投稿状況、Kindle単話版、診断フォームURL、SNS導線を管理表で一元管理し、運用だけは崩さない。

## 4. Season設計との接続

この出版同時展開方針も、Season設計の段階公開ルールの中で運用する。Season 1 は発見編であり、説明しすぎず、違和感と命名を優先する。したがって、Kindle単話版もSeason 1 では重い解説本にせず、**軽量な感染速報版**として設計する必要がある。[1] [2]

Season 2 以降になると、株、正式ビジュアル、図鑑カード、防衛要素などを少しずつ増やし、Kindleも単話速報版から、5話まとめ版、10話まとめ版、完全版へ段階的に育てるのが自然である。[1] [2]

| Season | 見せるもの | Kindleの位置づけ |
|---|---|---|
| Season 1 | ウイルス名、型、違和感 | 単話版「感染速報版」 |
| Season 2 | 株、図鑑要素、可視化 | 5話まとめ版、人気株再編集版 |
| Season 3 | ワクチン、防衛策 | 実用補強版、自己防衛連動版 |
| Season 4 | 分布、感染レベル、年鑑化 | 白書、年鑑、企業向け資料版 |

## 5. 初期拡散チャネルの再定義

今後の初期拡散チャネルは、YouTube Shorts、TikTok、Instagram Reels、8コマ漫画、Kindle単話版、診断フォームの六本柱で整理する。ここで特定媒体を主軸固定しないことが重要である。なぜなら、本企画では、どの媒体から入っても世界観へ到達し、別媒体へ再感染できる構造そのものが価値になるからである。

また、初期フェーズでは**公式サイトを必須にしない**。更新負荷を抑えつつ、発信源が固定されすぎないようにするためである。その代わり、Linktree、note固定記事、Kindle著者ページ、各SNSプロフィールなどを**簡易的な受け皿**として使い、表向きは自然発生感を保ちながら、必要な回遊だけを支える。

さらに、動画やSNSからKindleへ送客するだけでなく、**Kindleから動画・漫画・診断フォームへ、漫画から診断フォームへ、診断フォームからSNSへ**といった逆流導線も前提に置く。出版から火がつくケースも、漫画から火がつくケースも、診断結果の拡散から火がつくケースも想定し、各媒体を相互送客型で設計する。

| チャネル | 主な役割 | 初期運用の考え方 |
|---|---|---|
| YouTube Shorts | 最速の認知獲得 | 刺さる違和感を短時間で見せる |
| TikTok | コメントと共感拡散 | セリフと空気感で広げる |
| Instagram Reels | 世界観の見せ方強化 | ビジュアル統一感を保つ |
| 8コマ漫画 | 静止画での再編集 | 保存・再掲・別導線への橋渡しに使う |
| Kindle単話版 | 読める入口商品 | 面展開と導線形成に使う |
| 診断フォーム | 参加と再拡散 | 結果共有と観測導線に使う |
| 簡易受け皿 | 回遊の補助 | Linktree、note固定記事、著者ページ、SNSプロフィールで代替する |

## 6. Kindle単話版「感染速報版」の位置づけ

Season 1 の各EPは、まず**軽量なKindle単話版「感染速報版」**として、99円〜100円程度で順次出すことを基本想定とする。この単話版は利益商品ではない。認知拡大、シリーズ面展開、診断フォーム誘導、SNS導線のための**100円入口商品**として扱う。

感染速報版は、後からまとめ本に再利用されることを前提にする。したがって、単話版の時点から、表紙、本文構成、巻末導線、シリーズ内リンクを統一し、5話まとめ版、10話まとめ版へ無理なく再編集できるように設計しておく。

さらに、将来的な**英語版並行展開**を見据え、日本語版の画像、8コマ構成、ページ構成を流用しながら、英語タイトル、英語セリフ、英語図鑑ページへ差し替えられる状態を標準にする。つまり、Kindle単話版は単なる一冊単位ではなく、**日英差し替え可能なテンプレート商品**として設計する必要がある。

| 項目 | 感染速報版の考え方 |
|---|---|
| 価格帯 | 99円〜100円程度 |
| 役割 | 利益ではなく入口と認知拡大 |
| 主目的 | 面展開、シリーズ接触、診断フォーム誘導 |
| 想定内容 | 単話本文、軽い図鑑整理、巻末導線 |
| 言語設計 | 日本語版を基準に英語差し替え可能な構成 |
| 再利用先 | 5話まとめ版、10話まとめ版、完全版、英語版 |

## 7. Kindle導線設計

Kindle単話版には、必ず**動画・SNS・診断フォームへの導線**を入れる。逆に、動画の概要欄や固定コメント、SNS投稿、漫画投稿からもKindle感染速報版へ誘導する。これにより、各媒体が単独で終わらず、相互に感染を広げ合う設計になる。

運用上は、Kindle本文末尾に「動画を見る」「診断フォームへ進む」「次の感染速報版を見る」といった導線を置き、1冊の終わりが次の接点の入口になるようにするべきである。また、将来的な英語版では、日本語版と同じ画像・同じ8コマ構成を使いながら、タイトル、セリフ、図鑑説明、巻末導線文言のみを差し替えられるようにしておくべきである。

| 出発点 | 誘導先 | 意図 |
|---|---|---|
| Shorts | Kindle感染速報版 | 読みたい層を拾う |
| SNS投稿 | Kindle感染速報版 | 一覧性と面展開を強化する |
| Kindle感染速報版 | 動画 | 体験価値を補強する |
| Kindle感染速報版 | 診断フォーム | 参加導線へつなぐ |
| Kindle感染速報版 | 次話のKindle | シリーズ回遊を生む |
| 日本語版Kindle | 英語版Kindle | 海外向け横展開の導線基盤を作る |

## 8. 職場ウイルス・パンデミック戦略

今後は、動画から火がつくケースだけでなく、**出版、漫画、診断結果共有から火がつくケース**も同じ重みで想定する。たとえば、Kindleの表紙やタイトル設計が刺さって入口になる場合、8コマ漫画の保存や再投稿で共感が広がる場合、診断フォームの結果文がSNSで共有されて再流入を生む場合がありうる。

したがって、『職場ウイルス図鑑』では本・SNS・ショート動画・8コマ漫画・診断フォームのどれをメイン媒体と固定しない。中心は媒体ではなく、**職場ウイルス図鑑という世界観**であり、各媒体はその世界観を感染拡大させる経路である。

初期フェーズでは、ここに**サイトレス・パンデミック戦略**を重ねる。つまり、公式サイトを表に出して発信源を一本化するのではなく、各媒体がそれぞれ独立して観測される状態を優先する。視聴者や読者には「どこから出てきたかわからないが各地で観測される職場ウイルス」という印象を与えつつ、裏側では管理表で全導線を一元管理する。

| 感染経路 | 次の感染先 | 狙い |
|---|---|---|
| Kindleで知る | 動画へ移動 | 体験価値を補強する |
| 動画で知る | 漫画へ移動 | 保存・再読導線をつくる |
| 漫画で共感する | 診断フォームへ移動 | 参加行動へつなぐ |
| 診断フォーム結果を得る | SNSへ共有 | 再拡散を起こす |
| SNSで再接触する | Kindle・動画・漫画へ戻る | 回遊と反復接触を増やす |
| プロフィール導線で迷った人 | Linktreeやnote固定記事へ移動 | 公式サイトなしで最低限の受け皿を持つ |

この設計は、媒体ごとの単発成果を追うのではなく、**世界観への接触回数と回遊の連鎖**を重視する戦略である。よって、各媒体の評価も単独数値だけでなく、次媒体への送客率や戻り導線の強さと合わせて観測する必要がある。

## 9. 漫画・動画・出版の関係整理

『職場ウイルス図鑑』は、本、漫画、動画、診断を別々に作る企画ではない。**1つの原作台本から複数メディアへ同時展開する**ことが前提である。このとき、動画が原型、漫画が静止画翻訳、Kindleが読書体験化、フォームが参加導線という役割分担で考えるのが分かりやすい。

> 原作台本 → ショート動画 / 8コマ漫画 / Kindle単話版 / 診断フォーム
> 
> その後、反応と蓄積に応じて 5話まとめ版 → 10話まとめ版 → 年鑑・白書・企業向け資料へ接続する。

さらに重要なのは、これらを一方向の制作フローとしてだけでなく、**相互感染する循環フロー**として捉えることである。Kindleから動画へ、動画から漫画へ、漫画から診断フォームへ、診断フォームからSNSへ、SNSから再び各媒体へ戻る循環が成立して初めて、感染拡大型メディアミックス戦略が機能する。

この構造により、「売れた媒体だけを後から伸ばす」のではなく、「最初から複数入口と複数回遊を持つ」設計が成立する。

## 10. 5話・10話まとめ版への再利用方針

Kindle単話版は単発で終わらせず、**5話まとめ版**と**10話まとめ版**への再利用を最初から前提に置く。単話版で使ったタイトル、見出し、巻末導線、軽い図鑑整理は、将来のまとめ版で再編集可能な粒度にそろえておく必要がある。

同時に、将来的な英語版展開を考えると、再利用対象は日本語のまとめ版だけではない。日本語版の画像、8コマ構成、章立て、図鑑ページの枠組みを共通資産として保持し、テキストレイヤーのみ差し替えることで、英語単話版、英語5話版、英語10話版へ広げやすくしておくべきである。

5話まとめ版はSeason 1 前半の定着確認用、10話まとめ版はSeason 1 の一冊化用と考えると整理しやすい。単話版が入口、5話版がシリーズのまとまり、10話版が世界観の定着という役割分担になる。

| 版型 | 役割 | 想定用途 |
|---|---|---|
| 単話版 | 最小入口 | 認知拡大、導線形成 |
| 5話まとめ版 | 中間商品 | まとまりで読ませる |
| 10話まとめ版 | Season収録版 | 世界観の定着と保存 |
| 英語版 | 横展開商品 | 海外読者向け再編集 |
| 完全版 | 解説付き本命版 | 実用・分析・高単価化 |

## 11. 100円入口商品の位置づけ

100円前後のKindle感染速報版は、利益の最大化を目的にしない。むしろ、価格ハードルを下げることで、シリーズへの最初の接触率を上げ、読了後に他媒体へ送り返すことを重視する。

この意味で、感染速報版は「売上商品」というより、**読める名刺**であり、**シリーズ面展開の入口**であり、**観測導線つきのサンプル**である。単話ごとの冊数が増えること自体が、シリーズの存在感を強める効果も持つ。

## 12. EP単位の運用ルール

Season 1 の各EPでは、動画、SNS、漫画、Kindle単話版、診断フォームをできるだけ近いタイミングで設計し、相互誘導を持たせる。ここで重要なのは、同時公開の完全一致ではなく、**同時展開前提で素材設計を揃えること**である。

加えて、Kindle単話版は将来的な英語版並行展開を前提に、画像素材、8コマ構成、ページ順、図鑑見出し枠、巻末導線枠を共通化し、日本語版と英語版で差し替えるテキスト領域を明確に分ける。素材管理では、共通画像資産と日本語テキスト、日本語書影文言、英語テキスト、英語書影文言を分離して保持することが望ましい。

各EPの進行管理では、従来の「Kindle」一括管理では足りないため、今後は少なくとも**Kindle単話版**と**まとめ版再利用候補**を分けて管理する。さらに、必要に応じて英語版対応可否も備考で管理できるようにする。これにより、入口商品としての進捗と、後続商品としての進捗を混同せずに追える。

## 13. 最終目標

最終的には、『職場ウイルス図鑑』を、**SNSで共感され、動画で刺さり、Kindleで読まれ、漫画で広がり、診断フォームで参加され、年鑑や白書に蓄積され、企業向け診断や研修にも接続できるIP**へ育てることを目指す。

ここで重要なのは、どれか1媒体を頂点に置くことではない。中心は常に**『職場ウイルス図鑑』という世界観**であり、本・SNS・ショート動画・8コマ漫画・診断フォームはすべて、その世界観を広げる感染経路である。したがって、制作順序も評価軸も、媒体単位ではなく**感染連鎖の総体**として設計する必要がある。

また、初期フェーズでは公式サイトを前面に出さない。必要最小限の受け皿だけを置き、自然発生感を保ったまま感染経路を広げることを優先する。その後、企業向け診断、研修、年鑑化、資料化の段階へ進み、情報の集約拠点が必要になった時点で、はじめて公式サイトを立ち上げる。

ここで出版は、後から付け足す出口ではない。最初から感染を広げる入口であり、同時に観測を深める媒体である。この前提で制作順序を組み直すことが、今後の方針転換の中核になる。
